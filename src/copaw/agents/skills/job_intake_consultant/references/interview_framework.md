# 职位访谈参考框架

这份参考文档用于帮助 Agent 控制访谈深度，做到“问得少，但问得准”。

## 一、访谈优先级

### 高优先级：优先问

这些问题会直接影响搜人策略：

- 为什么现在要招这个岗位
- 这个人进来后 3 个月最重要的结果是什么
- 哪段经历必须做过，不能只靠潜力迁移
- 哪个筛选条件最不能放宽
- 哪类人表面匹配、实际高风险

### 中优先级：按需问

- 目标公司画像
- title 映射
- 薪资弹性
- 地域弹性
- 行业可迁移范围

### 低优先级：默认不主动问

- 详细组织结构
- 团队人数
- 常规福利
- 与搜人策略关系不大的背景资料

## 二、快速模式建议

适合 HR 日常开岗。

### 目标

- 尽量少追问
- 快速形成可执行画像

### 建议节奏

1. 用户自由表达需求
2. Agent 先归纳岗位主线
3. Agent 只补 1 到 3 个关键问题
4. 输出岗位画像和搜人建议

### 推荐追问示例

- “这个岗位最核心是补业务落地能力，还是补团队管理能力？”
- “如果只能保留 3 个必须项，你最看重哪 3 个？”
- “你更希望找完全同岗位的人，还是相近背景但能力可迁移的人？”

## 三、深挖模式建议

适合关键岗位、猎头委托岗位、画像长期模糊的岗位。

### 可以额外覆盖的维度

- 业务阶段
- 团队协作关系
- 候选人的入职说服点
- 面试关键信号
- 失败风险画像

### 注意

即使在深挖模式，也不要把对话变成问卷。

## 四、常见矛盾识别

### 矛盾类型

- 职责深度和薪资不匹配
- 要求“全都要”，但岗位实际优先级不清
- 希望候选人背景非常强，但城市或行业限制过窄
- 想招“即插即用”，又希望候选人来自完全不同赛道

### 推荐表达

- “如果这个条件不放宽，市场上可选样本会很少。”
- “我建议把 A 设为必须，把 B 从必须改成加分项。”
- “按当前描述，这个岗位更像高级经理 / 总监级画像，而不是普通经理画像。”

## 五、与搜人 skill 的交接格式

当要衔接 `duolie_talent` 或其他搜人环节时，建议最终沉淀成以下结构：

```markdown
## Search Brief

- 职位名称：
- 招聘目标：
- 必备项：
- 加分项：
- 可放宽项：
- 红线项：
- 关键词：
- 目标公司：
- 排除背景：
- 城市：
- 年限：
- 学历：
```

## 六、候选人推荐阶段的策略

这里强调的是 **猎头顾问的推荐策略**，不是修改底层搜人 skill 的能力边界。

### 默认策略

当职位画像还不够稳定时：

- 不要直接把第一批搜索结果列表展示给用户
- 先抽 3 到 5 个候选人样本
- 先读简历，再判断为什么像 / 为什么不像
- 利用样本继续和用户校正画像
- 等画像更稳定后，再做正式推荐

### 为什么这样做

因为用户一开始表达职位需求时，往往还不够完整，也未必准确。

与其让用户直接看一批噪音结果，不如先用样本帮助他确认：

- 他真正想要的人长什么样
- 哪些条件其实可以放宽
- 哪些条件必须收紧
- 当前搜索式为什么会带来 false positive

### 建议输出方式

不要直接说：

- “我搜到这些人，你看看”

建议说：

- “我先抽样看了前几位，目前更接近你需求的是 A 类，偏差较大的是 B 类。”
- “按这批样本看，你真正要的重点更像 X，而不是一开始说的 Y。”
- “下一轮我建议保留 A、B 两条，把 C 从必须改成加分项。”

### 什么时候可以直接给结果

满足以下任一条件时，可以直接展示更完整的候选人结果：

- 用户明确要求先直接看结果
- 职位画像已经非常稳定
- 已完成至少一轮样本校准
- 当前任务目标就是快速浏览样本库

## 七、停止追问的条件

满足以下任一条件时，应停止继续追问并转入输出：

- 已经拿到了足够形成初版画像的信息
- 继续提问不会明显改变搜人策略
- 用户表现出希望快速推进，而不是继续深挖
- 当前缺口可以在搜人或初筛阶段再校正
